# Настройка истории и аналитики загрузки гейтов

## Что это даёт
Единое хранилище = история коллег (153 536 рейсов) + наши ежедневные сборы.
Из него одной командой считается динамика загрузки гейтов: неделя-к-неделе,
месяц-к-месяцу, год-к-году.

## Структура в репозитории
```
data/
  history/
    history_ALL.csv        ← история коллег (положить сюда)
    history_ALL.parquet    ← появится после конвертации (быстрее, легче)
  daily/
    2026-06-01.csv         ← наши ежедневные сборы (бот кладёт сам)
    2026-05-31.csv
    ...
src/
  analytics.py             ← модуль аналитики
scripts_convert_to_parquet.py  ← разовая конвертация CSV→Parquet
```

## Шаги настройки (один раз)

1. **Загрузить историю.** Положи `history_ALL.csv` в `data/history/`
   (через веб GitHub: Add file → Upload files, перетащить файл в папку).

2. **Сжать в Parquet** (необязательно, но желательно — файл станет ~2 МБ
   вместо 11.5 МБ и будет читаться мгновенно). Способ A — локально:
   ```
   pip install pandas pyarrow
   python scripts_convert_to_parquet.py
   ```
   затем загрузить полученный `data/history/history_ALL.parquet` в репозиторий.
   Способ B — добавить шаг в GitHub Actions (см. ниже).

3. **Готово.** Ежедневные сборы бот уже кладёт в `data/daily/`. Аналитика
   видит оба источника автоматически.

## Как получать аналитику

```bash
# Сравнить два месяца по гейтам (динамика год-к-году):
python -m src.analytics --compare 2025-05 2026-05 --airport VKO

# Загрузка гейтов за последнюю неделю по терминалу:
python -m src.analytics --period week --airport SVO --terminal B

# За последний месяц по всему аэропорту:
python -m src.analytics --period month --airport DME
```

Вывод: по каждому гейту — число рейсов, доля внутри терминала, и при
сравнении — Δ в процентных пунктах (п.п.) между периодами.

## Методика доли (совпадает с методикой коллег, проверено)
> доля гейта = рейсов с гейта за период ÷ рейсов терминала за тот же период.
> Сумма долей всех гейтов терминала = 100%.

## Важные предостережения по данным коллег
- **Нули в старых периодах = «нет данных», а не «нет рейсов».** Часть гейтов
  VKO коллеги начали вести позже — в ранние месяцы у них 0. Модуль выводит
  предупреждение ⚠, если гейт имел 0 рейсов в одном из сравниваемых периодов.
  Сравнивай год-к-году только гейты, которые велись в оба периода.
- **История не сплошная**: у VKO пропуск янв–июль 2024, мелкие пробелы на
  праздниках. Это надо учитывать, а не молча усреднять.
- **Будущие даты** (план, не факт) в истории помечены `is_future` и
  автоматически отсекаются из аналитики.
- **Терминал C у SVO** в части истории велся по группе гейтов, а не по
  отдельному — там доля считается по группе.
